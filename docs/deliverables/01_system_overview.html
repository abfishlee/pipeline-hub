Unified Data Pipeline Platform

농축산물 가격 데이터를 다채널에서 실시간 수집하고, AI 표준코드로 정규화하여 신뢰 가능한 가격 마스터로 통합·서비스하는 데이터 플랫폼.

1. 프로젝트 한 문장 요약

“데이터를 많이 넣는 시스템이 아니라, 어떤 가격 데이터가 언제 어디서 들어와 어떤 규칙으로 표준화되어 어떤 상품 마스터에 반영되었는지 끝까지 추적 가능한 플랫폼.”

2. 시스템 목표 + 비즈니스 가치

실시간성

수집 후 1분 이내 mart 반영. 10만~30만 rows/일, 영수증 OCR 1,000페이지/일 처리.

정합성

채널 7종의 이질적 데이터를 동일 표준코드로 정규화. confidence ≥ 0.85 만 자동 반영, 미달은 사람이 검수.

추적 가능성

raw → stg → mart 전 단계 lineage. 모든 SQL 실행 audit, 배포 이력에 그래프 스냅샷 동봉.

운영 효율

운영자가 코드 없이 Visual ETL Designer 로 파이프라인 설계 → SQL Studio 에서 dry-run + 승인 → cron 자동 실행.

외부 서비스화

Public API 로 가격 데이터를 외부 소비자에게 제공 (Phase 4 — API Key + Rate Limit + RLS).

운영 비용 통제

Naver Cloud 친화 — Object Storage / Cloud DB / NKS 활용. 자체 Kafka 도입은 트래픽 500K/일 초과 시에만 (조건부).

3. 4계층 데이터 모델

데이터는 다음 4계층을 좌→우로 흘러가며, 각 계층은 책임이 분리된다. 잘못된 데이터가 흘러들어도 raw 가 보존되어 있어 마스터 재처리 가능.

3.1 계층별 책임 + 보존 정책

계층책임보존핵심 보장
RAW 채널 원천 형태 그대로 보존 (JSON/IMG) 13개월 → Object Storage cold (Phase 4) idempotency_key, content_hash 중복 방지
STG 파싱 + 표준화 후보 + DQ 검증 3개월 (DQ 통과만 mart 반영) std_code 매핑 + confidence 게이트
MART 표준 마스터 + 가격 사실 + 일별 집계 영구 (월 파티션 RANGE) UPSERT, ON CONFLICT 키 + raw_object_id 추적
SERVICE 내부/외부 API + Web Portal — JWT + RBAC + (Phase 4) Public API Key + Rate Limit

4. 핵심 기능 + 기능별 기술 스택

4.1 다채널 수집 게이트웨이

FastAPI 0.115 — 비동기 endpoint 3종 (/v1/ingest/{api,file,receipt}). raw 보존 + idempotency_key 강제. Phase 1 핵심.

FastAPI SQLAlchemy 2.0 async asyncpg / psycopg3 argon2id

4.2 워커 + 메시지 브로커

Dramatiq + Redis Streams — 5종 worker (outbox / ocr / transform / price_fact / db_incremental / crawler). idempotent_consume 패턴 + Consumer Group fan-out.

Dramatiq 1.17 Redis Streams outbox + content_hash

4.3 OCR — 영수증 / 전단

CLOVA OCR + Upstage 폴백. confidence ≥ 0.85 자동, 미만은 Crowd 검수 큐. 영수증 개인정보 마스킹 필수.

CLOVA OCR Upstage circuit breaker

4.4 AI 표준화 (3단계 매칭)

pg_trgm 0.7 → HyperCLOVA 임베딩 0.85 → Crowd. pgvector(1536) IVFFLAT. 시맨틱 + lexical 폴백 (ADR-0005).

pg_trgm HyperCLOVA pgvector

4.5 Visual ETL Designer

React Flow 12 캔버스 — 7노드 drag-and-drop / 엣지 연결 / 위치 자동 저장. PUBLISH 시 그래프 freeze + 새 PUBLISHED 워크플로 자동 생성 (ADR-0007/0009).

@xyflow/react 12 React 18 + TS strict TanStack Query Tailwind CSS

4.6 SQL Studio (sandbox + 승인)

sqlglot AST + read-only TX + LIMIT 자동 부착 + ROLLBACK. DRAFT→PENDING→APPROVED 상태머신 + self-approval 차단 (ADR-0008).

sqlglot 26 PG SET LOCAL transaction_read_only audit.sql_execution_log

4.7 자체 DAG 실행기 + SSE

Kahn 토폴로지 정렬 + 노드 actor 디스패치. Pub/Sub → fetch-event-source 로 실시간 노드 상태 반영 (≤500ms 목표).

Kahn topo-sort Redis Pub/Sub Server-Sent Events

4.8 배포 이력 + diff

PUBLISH 마다 node/edge 스냅샷 + 변경 요약 jsonb 저장. 임의 두 워크플로 A→B 방향 diff (added/removed/changed).

JSONB snapshot node_key 기준 diff

4.9 스케줄 / Backfill / 재실행

croniter 5필드 검증 + 날짜 범위 backfill (멱등) + ancestors SUCCESS 시드 후 특정 노드부터 재실행.

croniter PARTITION BY RANGE 멱등 (workflow_id, run_date)

4.10 관제 + 감사

Prometheus + Grafana + Loki + Sentry. structlog JSON 라벨링, request_id 전파, audit 3종 (access / SQL / download).

Prometheus Grafana Loki + Promtail Sentry (PII scrub)

5. 데이터 파이프라인 프로세스 (10단계)

#단계처리 노드핵심 동작
①수집FastAPI /v1/ingest/{api,file,receipt} 채널 원천 → idempotency_key + raw 영속
②raw 영속raw.raw_object + Object Storage JSON inline / IMAGE 큰 파일은 S3 호환 버킷
③outbox 발행run.event_outbox + Redis Streams publisher 가 트랜잭션 정합 보장
④파싱SOURCE_API / OCR / CRAWLER worker 채널별 worker 가 raw 를 정형 row 로 변환
⑤표준화transform worker (3단계) trgm 0.7 → embedding 0.85 → crowd 폴백
⑥중복 제거DEDUP 노드 (sku, captured_at) 키 DISTINCT ON
⑦DQ 검증DQ_CHECK 노드 row_count / null_pct / unique / custom_sql
⑧mart 적재LOAD_MASTER 노드 ON CONFLICT UPSERT (월 RANGE 파티션)
⑨알림NOTIFY 노드 (outbox) Slack / Email / Webhook (Phase 4 발송)
⑩서비스화/v1 (내부) + /public/v1 (Phase 4) JWT/RBAC + (Ph4) API Key + Rate Limit
