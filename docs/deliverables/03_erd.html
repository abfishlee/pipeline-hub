Entity Relationship Diagram (ERD)

Unified Data Pipeline Platform — Phase 3 종료 시점 (migration 0001~0020 적용 후) DB 스키마 전체. 8 schema · 30+ table · 47 foreign key.

버전  Phase 3 종료 (2026-04-26) 대상  DBA / 데이터 모델러 / 아키텍트 관련 ADR  0001 / 0003 / 0006 / 0009

1. Schema 8종 — 책임 분리

데이터는 좌→우로 흘러가며, 각 schema 는 명확한 책임. 한 schema 안의 변경이 다른 schema 에 영향 주지 않도록 FK 가 좁게 설정되어 있다 (예: mart.* 가 raw.* 에 직접 의존하지 않음 — stg.* 를 거쳐 흐름).

ctl — 제어 / 메타

사용자, 역할, API 키, 데이터 소스, 커넥터. 모든 schema 가 이쪽을 참조.

raw — 원천 보존

채널 원천 형태 그대로 (JSON / IMAGE / RECEIPT / HTML). idempotency 보장.

stg — 표준화 후보

raw → 정형 row 1차 파싱 + std_code 매핑. DQ 게이트 통과 row 만 mart 로.

mart — 통합 마스터 + 팩트

standard_code, retailer/seller/product master, price_fact, 일별 집계.

wf — Visual ETL Designer

워크플로 정의 (DRAFT/PUBLISHED), 노드/엣지, 배포 이력, SQL 자산 + 버전 + 승인.

run — 실행 이력

pipeline_run / node_run, ingest_job, outbox, dead_letter, crowd_task.

audit — 감사 / 컴플라이언스

access_log (PARTITIONED), sql_execution_log, download_log.

dq — 데이터 품질

DQ_CHECK 노드 결과. row_count / null_pct / unique / custom_sql.

3. Schema 별 상세

ctl — 제어 / 메타

모든 schema 가 의존하는 코어. 사용자/역할/API 키/데이터 소스. seed_admin.py 로 초기 ADMIN 1명 시드.

테이블주요 컬럼비고
app_useruser_id PK, login_id UQ, password_hash (argon2id), is_active로그인 주체
rolerole_id PK, role_code UQ5종: ADMIN/APPROVER/OPERATOR/REVIEWER/VIEWER
user_role(user_id, role_id) PK 두 개의 FK다대다 매핑
api_keyapi_key_id PK, prefix, hashPhase 4 Public API 발급용
data_sourcesource_id PK, source_code UQ, source_type, watermark JSONBAPI/DB/CRAWL/OCR 채널
connectorconnector_id PK, source_id FK외부 시스템 인증 정보

raw — 원천 보존

모든 채널이 채널 원천 형태 그대로 적재. idempotency_key + content_hash dedup (ADR-0003) 으로 같은 데이터가 두 번 들어오지 않게.

테이블주요 컬럼비고
raw_object PARTITIONED (raw_object_id, partition_date) PK, source_id FK, object_type, payload JSONB, object_key 월 RANGE 파티션. 큰 payload 는 Object Storage object_key 만 저장
content_hash_index source_id FK, content_hash (SHA-256), partition_date 같은 raw 내용 중복 방지
raw_web_page source_id FK, url, parser_version, html bytes 크롤 HTML 전용 (Phase 2.2.8)
ocr_result raw_object_id, provider (CLOVA/Upstage), confidence OCR worker 결과 (Phase 2.2.4)
db_snapshot source_id FK, before/after JSONB, op_kind DB-to-DB CDC 시 변경 전/후 모두

stg — 표준화 후보

raw 가 worker 에 의해 정형 row 로 1차 파싱된 결과. std_code 매핑 + confidence 게이트 통과 row 만 mart 로 흐름.

테이블주요 컬럼비고
price_observation obs_id PK, source_id FK, std_code FK, retailer_code, store_name, product_name_raw, price_krw, weight_g, observed_at, std_confidence 채널 7종이 모두 이리로 수렴
standard_record source_id FK, embedding VECTOR(1536) 표준화 후보 + 임베딩 (Phase 2.2.5)

mart — 통합 마스터 + 팩트

표준코드, retailer/seller/product master, price_fact (월 RANGE 파티션), 일별 집계. std_code 가 식품 분류의 root key.

테이블주요 컬럼비고
standard_code std_code PK, name_ko, category, embedding VECTOR(1536) IVFFLAT cosine 인덱스. 표준화 매칭 후보
retailer_master retailer_id PK, retailer_type (MART/SSM/LOCAL/ONLINE/TRAD_MARKET/APP) 유통사 단위
seller_master seller_id PK, retailer_id FK 매장 단위 (예: 이마트 용산점)
product_master product_id PK, std_code FK, canonical_name, weight_g 표준 상품 마스터
product_mapping mapping_id PK, product_id FK, retailer_id FK, retailer_product_code, verified_by FK 유통사별 sku ↔ product_id 매핑
price_fact PARTITIONED (price_id, partition_date) PK, product_id FK, seller_id FK, price_krw, observed_at 가격 사실. 월 파티션 (2026-04 ~ 2026-12 미리 생성)
price_daily_agg (agg_date, std_code, retailer_id, region_sido) PK, min/avg/max/median/count 일별 집계 (00:30 자동 재계산)
master_entity_history (entity_id, version) PK master 변경 이력 (Phase 4)

wf — Visual ETL Designer (Phase 3)

워크플로 정의 (DRAFT/PUBLISHED), 노드/엣지 그래프, 배포 이력 + 그래프 스냅샷, SQL 자산 + 5상태 라이프사이클.

테이블주요 컬럼비고
workflow_definition workflow_id PK, (name, version) UQ, status, schedule_cron, schedule_enabled DRAFT/PUBLISHED/ARCHIVED 3상태
node_definition node_id PK, workflow_id FK, node_key (UQ workflow_id+key), node_type (7종), config_json JSONB, position NOOP/SOURCE_API/SQL_TRANSFORM/DEDUP/DQ_CHECK/LOAD_MASTER/NOTIFY
edge_definition edge_id PK, workflow_id FK, from_node_id FK, to_node_id FK self-loop 금지 (CHECK 제약)
pipeline_release release_id PK, workflow_name, version_no, source_workflow_id FK, released_workflow_id FK, change_summary JSONB, nodes_snapshot, edges_snapshot PUBLISH 시 1행 INSERT (ADR-0009)
sql_query sql_query_id PK, name UQ, owner_user_id FK, current_version_id FK (자기참조 우회) SQL 자산 메타
sql_query_version v_id PK, sql_query_id FK, version_no, status (5종), parent_version_id FK (자기참조), referenced_tables DRAFT/PENDING/APPROVED/REJECTED/SUPERSEDED

run — 실행 이력

pipeline_run (월 파티션) + node_run 상태머신, ingest_job, outbox 패턴, dead_letter, crowd_task placeholder.

테이블주요 컬럼비고
pipeline_run PARTITIONED (pipeline_run_id, run_date) PK, workflow_id FK, status, triggered_by FK 월 RANGE 파티션
node_run node_run_id PK, (pipeline_run_id, run_date) FK 합성, node_definition_id FK, status, attempt_no, output_json JSONB PENDING→READY→RUNNING→SUCCESS/FAILED/SKIPPED/CANCELLED
ingest_job job_id PK, source_id FK, job_type (ON_DEMAND/SCHEDULED/RETRY/BACKFILL), parameters JSONB 채널 수집 작업 큐
event_outbox event_id PK, aggregate_type/id, event_type, payload JSONB, status outbox 패턴 (ADR-0003) — Redis Streams 로 이송
processed_event (event_id, consumer_name) PK idempotent consumer marker
dead_letter dl_id PK, payload JSONB, replayed_by FK 실패 메시지 보관 + 재처리
crowd_task crowd_task_id PK, raw_object_id, reason, status, payload JSONB, reviewed_by FK Phase 2 placeholder → Phase 4 정식 (ADR-0006)

audit — 감사 / 컴플라이언스

모든 사용자 활동 + SQL 실행 + 다운로드 흔적. Phase 1.2.10 의 미들웨어 자동 적재.

테이블주요 컬럼비고
access_log PARTITIONED (log_id, occurred_at) PK, user_id FK, api_key_id FK, method, path, status_code, duration_ms, ip INET, request_id 월 RANGE 파티션. 모든 HTTP 요청
sql_execution_log sql_log_id PK, user_id FK, sql_query_version_id FK, sql_text, sql_hash, execution_kind (6종), status, row_count VALIDATE/PREVIEW/EXPLAIN/SANDBOX/APPROVED/SCHEDULED
download_log download_id PK, user_id FK, resource_kind, resource_ref, byte_count 외부 다운로드 흔적

dq — 데이터 품질

DQ_CHECK 노드의 검사 결과. pipeline_run / node_run 와 loose link (FK 미설정 — 파티션 CASCADE 회피).

테이블주요 컬럼비고
quality_result quality_result_id PK, pipeline_run_id, node_run_id (FK 없음), target_table, check_kind, passed, severity (INFO/WARN/ERROR/BLOCK), details_json row_count_min / null_pct_max / unique_columns / custom_sql

4. 핵심 정책 + 인덱스

정책대상이유
RANGE 파티션 (월별) raw.raw_object, run.pipeline_run, mart.price_fact, audit.access_log 대량 시계열 데이터 — 13개월 후 cold archive 가 쉬워야 (Phase 4)
pgvector(1536) IVFFLAT mart.standard_code.embedding, stg.standard_record.embedding HyperCLOVA 임베딩 cosine top-k 매칭 (Phase 2.2.5)
pg_trgm (trigram) mart.standard_code.name_ko 표준화 1단계 lexical similarity ≥ 0.7
UNIQUE (workflow_definition.name, version) wf.workflow_definition 같은 name 안에서 version 자동 +1 안전장치 (ADR-0009)
CHECK 제약 edge_definition (self-loop 금지), workflow_definition.status, node_run.status, ... 상태머신 강제 (DB 레벨 안전망)
BRIN 인덱스 mart.price_fact (시계열 append-heavy) 인덱스 크기 1/100, 시계열 쿼리에 충분
Partial 인덱스 wf.workflow_definition (schedule_enabled=TRUE AND status=PUBLISHED) Phase 4 Airflow polling 가속
ON CONFLICT (UPSERT) mart.price_fact, mart.product_master LOAD_MASTER 노드의 idempotent 적재
FK use_alter (자기참조 사이클) wf.sql_query.current_version_id ↔ wf.sql_query_version SQLAlchemy create_all 시 FK 사이클 풀기
